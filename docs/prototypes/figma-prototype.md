## 方向

用户选择融合方向 `1 高效运营型` + `2 社区生鲜型`：

- PC 后台：以高效运营为主，强调订单处理、筛选、批量发货、库存预警和权限边界。
- 小程序：以社区生鲜为主，强调套餐剩余、重量进度、真实菜品图、默认地址和确认下单。

## 设计说明

会员用户和后台用户在原型中保持分离：

- `会员用户` 用于管理 C 端用户资料、套餐、地址和订单。
- `后台用户与权限` 用于管理系统登录账号、角色和操作权限。
- `后台登录` 面向运营管理员和超级管理员，使用后台用户账号登录，并在登录后按角色展示菜单和操作权限。
- `后台两级导航` 采用工作台 + 订单管理、会员管理、套餐管理、任务管理、系统管理分组，二级菜单承载具体页面；菜单支持展开 220px 和折叠 72px 两种状态，折叠后保留图标入口并在悬停时显示一级/二级提示。
- `侧栏收缩按钮` 固定在侧栏右上边界，使用“面板 + 箭头”图标按钮，不放在菜单列表内部。
- `二级菜单收缩` 已补充系统管理示例：一级分组标题带 chevron 图标，点击可收起后台用户、操作日志、系统设置等二级菜单；行内不显示“展开/收起”文字，当前路由所在分组默认保持展开，手动状态可记忆。
- `后台用户区` 顶部右侧不只显示用户名，改为头像、姓名、角色、数据范围和下拉菜单组合；下拉菜单预留个人资料、切换角色、操作日志、退出登录。
- `后台字号` 已提高可读性：菜单一级分组标题使用 15px，关键一级入口使用 16px，二级项使用 15px，表格和正文基础字号不低于 13px；小程序避免 11px 以下文字。
- `后台工作弹窗` 用于详情、编辑、新建，支持拖拽、全屏和伸缩，替代右侧详情抽屉和独立详情页；右上角全屏控制使用四角最大化图标，不使用文字按钮。
- `小程序登录` 面向会员用户，参考瑞幸账户登录页的“大视觉 + 双按钮 + 底部协议”结构，使用蔬菜照片作为主视觉，保留微信一键登录和手机号安全登录；无套餐引导不放在登录页。
- `小程序导航` 底部 Tab 固定为：首页、我的；不设置单独的预订、订单、套餐菜单。首页就是预订入口，订单和套餐从“我的”页入口进入。
- `小程序预订` 参考瑞幸菜单页的高效结构，改为左侧分类 + 右侧菜品列表 + 底部已选篮子；顶部只保留短状态、套餐额度、默认地址和截单时间，避免解释性内容压住操作。
- `首页额度展示` 套餐卡负责展示已选、总额度和剩余；底部篮子只展示已选菜品重量和提交按钮，不重复展示剩余额度。
- `无套餐状态` 首页仍可浏览今日菜品，但加菜和提交入口禁用，提示“暂无有效套餐，不能下单”；购买套餐入口和微信支付文案仅做预留，当前不进入真实支付。
- `修改已预订内容` 在截单前开放，进入修改态后保留原订单提示、预填已选菜品，底部主按钮改为“保存修改”，确认页展示原订单和本次变化。
- `文字溢出` 已对提交确认和修改预订页做短句处理，底部篮子、提示卡和地址行需要按小程序窄屏宽度预留换行空间。
- `套餐与我的` 参考瑞幸的会员身份卡片感，但内容改为蔬菜套餐权益、剩余额度、今日预订、常用服务和消耗记录；入口命名为“套餐”和“订单”，套餐页预留购买/续费入口，不做咖啡式优惠券堆叠。

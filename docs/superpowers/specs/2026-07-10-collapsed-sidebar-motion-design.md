# 收缩侧栏与二级菜单动画设计

## 目标

管理后台采用一级分组图标的收缩侧栏。收缩后保留工作台、订单管理、会员管理、套餐管理、任务管理、系统管理六个一级图标；悬停或聚焦一级图标时，展示该分组全部二级菜单及对应图标。

本次重点解决：

- 系统管理二级菜单较长，靠近视口底部时内容被遮挡或误以为未展示。
- 侧栏收缩、展开目前只有宽度突变感，文字、图标和浮层缺少协调动画。
- 二级菜单需要保持可点击，同时任何时刻只能出现一个浮层。

## 交互设计

### 收缩状态

- 侧栏宽度为 `72px`，只显示六个一级分组图标。
- 当前页面所属分组保持绿色选中态。
- 悬停或键盘聚焦一级图标，打开对应二级菜单浮层。
- 浮层展示分组名称、全部二级菜单图标和文字。
- 鼠标从一级图标移动到浮层期间保持打开；离开整个触发区域后关闭。
- 移入另一个一级图标时，旧浮层退出后切换到新浮层，任何时刻最多显示一个。
- 点击二级菜单后立即关闭浮层并完成页面跳转。
- 按 `Escape` 关闭当前浮层。

### 系统管理长菜单

- 浮层以视口为边界定位，不固定向下展开。
- 根据触发图标位置和浮层实际高度计算顶部位置，保证上下至少保留 `12px` 安全间距。
- 内容超过可用高度时，浮层内部纵向滚动，不让菜单超出屏幕。
- 该逻辑适用于所有一级分组，不对“系统管理”写死特殊位置。

### 展开状态

- 侧栏宽度为 `220px`，显示品牌文字、一级分组名称和已展开的二级菜单。
- 收缩时先淡出文字，再缩小侧栏；展开时先扩展侧栏，再淡入文字，避免文字挤压和闪烁。
- 用户原有的分组展开状态继续由现有本地偏好设置保存。

## 动画方案

新增 `motion` 作为管理端依赖，使用 `motion/react`：

- `motion.aside`：侧栏宽度在 `72px` 和 `220px` 之间过渡。
- `AnimatePresence`：品牌文字、菜单文字和二级菜单浮层的进入与退出动画。
- 浮层动画：透明度 `0 → 1`、水平位移 `6px → 0`、缩放 `0.98 → 1`。
- 侧栏与浮层动画时长控制在 `180–220ms`，使用柔和的 ease-out 曲线。
- 使用 `MotionConfig reducedMotion="user"` 尊重系统“减少动态效果”设置。

动画只作用于 `transform`、`opacity` 和必要的侧栏宽度，不阻塞鼠标点击和键盘操作。

## 组件与状态

继续在 `AdminShell` 中维护单一的 `collapsedOpenGroup`，不为每个分组创建独立布尔状态。

新增两个小型、可独立测试的职责：

1. 浮层定位函数：输入触发器位置、浮层高度和视口高度，返回经过上下边界限制的 `top`。
2. 收缩菜单浮层组件：负责渲染当前分组、测量高度、执行 Motion 进入退出动画和处理滚动。

不修改导航数据结构、权限判断、路由参数或后台接口。

## 异常与边界

- 当前分组没有可访问的二级菜单时，不渲染图标和浮层。
- 视口高度较小时，浮层使用最大高度和内部滚动。
- 页面切换、侧栏展开或布局模式切换时，立即清理当前收缩浮层状态。
- 动画依赖加载失败不会影响路由和菜单可点击性；菜单结构仍由 React 正常渲染。

## 验证

- 单元测试覆盖浮层顶部位置的正常、顶部越界和底部越界三种情况。
- 使用测试覆盖同一时间只打开一个收缩浮层、移入浮层保持、离开关闭、点击关闭。
- 管理端 TypeScript 检查、相关 Vitest 和生产构建通过。
- 部署后分别在测试 `8202`、生产 `8102` 验证：收缩展开动画、系统管理完整菜单、视口底部避让和二级菜单点击跳转。
